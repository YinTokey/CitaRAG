# Parsing and Chunking Strategy

## Overview
This document outlines the advanced file parsing and chunking strategy used in CitaRAG to ensure high-quality retrieval and precise citation.

## 1. Element-Based Parsing
Instead of converting everything to plain text, we use **Jsoup** to parse XHTML (generated by Tika) into typed `DocumentElement` objects.

### Element Types
-   **TITLE**: Headers (H1-H6). Critical for hierarchy.
-   **NARRATIVE_TEXT**: Standard paragraphs.
-   **TABLE**: Preserved as atomic units with full HTML structure.
-   **LIST_ITEM**: Bullets/Numbering.

## 2. Context-Enriched Component Chunking
We employ a **Structure-Aware** strategy that prevents context loss in large documents.

### Hierarchy Tracking
The `ChunkingService` maintains a stack of active headers (e.g., `1. Business > 1.1 Risks`).

### Context Injection
Every chunk receives a **Context String** prepended to its text.
-   *Format*: `[Context: Grandparent Header > Parent Header]`
-   *Benefit*: A paragraph about "pricing" in a "Competitors" section will be indexed as `[Context: Competitors] pricing...`, ensuring it matches queries for "competitor pricing" even if the word "competitor" isn't in the immediate paragraph.

### Atomic Tables
Tables are never split mid-row. They are treated as single, atomic chunks to preserve row/column relationships.

## 3. Metadata Enrichment (AI-Powered)
To enable **precise citation**, every chunk is passed through an LLM (`gpt-4o-mini`) to generate structured metadata (`ChunkMetadata`).

### Generated Fields
1.  **Summary**: 1-2 sentence overview.
2.  **Keywords**: High-value entities for keyword search.
3.  **Hypothetical Questions**: Questions the chunk answers (optimizes RAG retrieval).
4.  **Table Insights**: Natural language interpretation of table trends.
5.  **Citation Context**: A generated sentence suitable for citing this chunk.
6.  **Key Statistics**: Extracted numerical data points.

## Data Flow
1.  **Upload**: File -> `FileParserService`
2.  **Parse**: `DocumentElement` stream (Titles, Text, Tables)
3.  **Chunk**: `ChunkingService` processes elements -> Context Injection -> Split/Group
4.  **Enrich**: `MetadataEnrichmentService` -> GPT-4o-mini -> `ChunkMetadata`
5.  **Save**: JSON output to `data/chunks/`
